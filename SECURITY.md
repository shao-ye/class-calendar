# 安全政策

## 支持版本

当前项目处于早期阶段，安全修复优先面向 `master` 分支。

## 报告漏洞

请不要在公开 Issue 中披露可被利用的漏洞细节、密钥、真实账号或个人隐私数据。

你可以通过 GitHub 私密安全报告功能提交漏洞；如果仓库暂未启用该功能，请先创建一个不包含利用细节的 Issue，说明希望私下沟通安全问题。

报告时建议包含：

- 受影响的功能或接口
- 复现条件和影响范围
- 是否涉及认证绕过、数据泄露、越权访问或密钥暴露
- 建议的修复方向

## 密钥处理

本项目使用 Cloudflare Pages、D1、R2 和 Worker。部署者应自行管理并定期轮换：

- `SESSION_SECRET`
- `SYNC_KEY`
- Cloudflare API Token
- D1 database ID
- R2 bucket 配置
- 登录用户密码哈希

如果任何密钥、口令哈希或真实数据曾经进入 Git 历史，请在公开仓库前清理历史并轮换相关凭据。
