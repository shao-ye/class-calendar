# 孩子每日课程记录日历 —— 开发设计文档

> 配套：[需求设计文档](需求设计文档.md)、[项目架构文档](项目架构文档.md)、[部署说明文档](部署说明文档.md)

## 一、技术栈

| 层 | 选型 |
| --- | --- |
| 前端 | Vue 3（`<script setup>`）+ Vue Router 4 + Vite 6 |
| UI | 自绘 CSS（无第三方组件库），移动端优先 |
| 后端 | Cloudflare Pages Functions（Workers 运行时，ESM） |
| 数据库 | Cloudflare D1（SQLite） |
| 对象存储 | Cloudflare R2 |
| 密码/会话 | Web Crypto（PBKDF2 + HMAC-SHA256） |
| 定时任务 | 独立 Worker + Cron Triggers |

## 三、数据模型

完整 DDL 见 `schema.sql` 与 [需求设计文档 5.4](需求设计文档.md)。核心表：

| 表 | 关键字段 | 说明 |
| --- | --- | --- |
| `users` | username, password_hash, display_name | 用户 |
| `auth_identities` | user_id, provider, provider_uid, union_id | 登录方式解耦（为微信预留） |
| `courses` | user_id, name, color, default_start, default_duration, weekdays, skip_holiday, track_hours | 课程库（含排课规则与课时开关） |
| `course_packages` | course_id, total_hours, opening_used, purchase_date, amount, status | 课时包（循环统计） |
| `records` | user_id, date, course_id, start/end_time, duration, status, package_id, consumed_hours, note | 上课记录 |
| `photos` | record_id, r2_key | 照片元数据 |
| `holidays` | date(PK), name, is_workday | 全局假日（is_workday=1 为调休补班） |

约定：`weekdays` 存 `"2,4"`（1=周一…7=周日）；`status` ∈ `attended|leave|reschedule`；`course_packages.status` ∈ `active|done`。

## 四、接口清单（已实现）

所有 `/api/*`（除 `auth/login`、`auth/logout`）均需会话，并按 `user_id` 隔离。

### 认证
| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/auth/login` | body `{username,password}` → 校验 + 下发 `session` Cookie |
| POST | `/api/auth/logout` | 清除 Cookie |
| GET | `/api/auth/me` | 返回 `{id,username,displayName}`（未登录 401） |

### 课程
| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/courses` | 课程列表；统计课时的课附 `activePackage{total,used,remaining,...}` |
| POST | `/api/courses` | body `{name,color,defaultStart,defaultDuration,weekdays[],skipHoliday,trackHours}`；重名 409 |
| PUT | `/api/courses/:id` | 同上字段更新 |
| DELETE | `/api/courses/:id` | 有记录则软停用(`is_active=0`)并回传 `softDeleted`，否则硬删 |

### 课时包
| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/courses/:id/packages` | 该课全部包（含 used/remaining） |
| POST | `/api/courses/:id/packages` | 续费/新增；旧 active 自动归档 done；body `{total,used,purchaseDate,amount}` |
| PUT | `/api/packages/:pid` | 改总课时/已上/日期/金额/状态；`used` 会折算回 `opening_used` |
| DELETE | `/api/packages/:pid` | 删包（关联记录 `package_id` 置空） |

### 记录
| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/records?month=YYYY-MM` | 某月（日历渲染，不含照片） |
| GET | `/api/records?date=YYYY-MM-DD` | 某天（含 `photos:[id]`） |
| GET | `/api/records?from=&to=` | 日期区间（周视图） |
| POST | `/api/records` | 新增；到课且课程统计课时 → 自动绑定生效包并扣 `consumedHours`（默认 1） |
| PUT | `/api/records/:id` | 更新；重算课时绑定 |
| DELETE | `/api/records/:id` | 删除（连带删 R2 照片对象 + 元数据） |

### 照片
| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/photos` | multipart `recordId + file`；存 R2 + 写元数据 |
| GET | `/api/photos/:id` | 鉴权后从 R2 读流返回图片 |
| DELETE | `/api/photos/:id` | 删 R2 对象 + 元数据 |

### 假日
| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/holidays?year=YYYY` | 读某年假日（前端排课用） |
| POST | `/api/holidays/sync` | 手动从外部源同步（当年+次年），返回 `{count}` |

## 五、关键实现逻辑

### 5.1 密码与会话（`functions/utils/auth.js`）
- 哈希：`pbkdf2$迭代$saltB64$hashB64`，PBKDF2 / SHA-256 / 10 万次 / 16B 盐 / 256bit。
- 会话：`base64url(payload).hmacSHA256(payload, SESSION_SECRET)`，payload 含 `{uid, exp}`；Cookie `session`，HttpOnly + SameSite=Lax + 30 天。
- `_middleware.js` 校验签名与过期，注入 `data.uid`。

### 5.2 课时核算（`functions/utils/package.js`）
- 已用 = `opening_used + Σ(records.consumed_hours where package_id=本包 and status='attended')`。
- 剩余 = `total_hours − 已用`；`activePkg` 取最新 `status='active'`。
- PUT 包时若传 `used`（界面的"已上"），服务端用 `used − 记录消耗` 反推 `opening_used`，使手动校正与自动累加并存。

### 5.3 记录与课时联动（`functions/api/records*`）
- 新增/更新：到课 + 课程 `track_hours` → `package_id=生效包`、`consumed_hours`；非到课不绑定。
- 因"已用"是实时 SUM，改状态为请假即自动不计入，无需手动回退。

### 5.4 自动排课投影（`src/views/Calendar.vue` `plannedByDate`）
- 从今天起按 `weekdays` 向后枚举；`skip_holiday` 时跳过 `holidays` 中 `is_workday=0` 的放假日。
- 数量：统计课时的课=`floor(剩余课时)`；其余=固定 30 个。
- 已有真实记录的当天不重复显示；"确认到课"→ `POST /api/records`。

### 5.5 照片（`src/views/Calendar.vue` + `functions/api/photos*`）
- 前端 canvas 压缩（最长边 1280 / JPEG 0.8）后 multipart 上传；缩略图用 `<img src="/api/photos/:id">`。

### 5.6 假日同步（`worker-holidays/src/index.js`）
- `scheduled` 与受 `SYNC_KEY` 保护的 `fetch` 共用 `syncHolidays`：拉 `{year}.json` → `INSERT OR REPLACE holidays`（`isOffDay?0:1`），当年+次年，单年失败不影响其它年。

## 六、本地开发

```bash
npm install
npm run db:init      # 本地 D1 建表
npm run db:seed      # 默认用户 parent/123456
# 两个终端：
npm run server       # wrangler pages dev dist --port 8788（Functions + D1 + R2）
npm run ui           # vite(5173)，/api 代理到 8788，热更新
```
访问 `http://localhost:5173`。改密码：`node scripts/gen-hash.mjs <新密码>` 后替换 `seed.sql` 重跑 `db:seed`。

> 注意：`wrangler.toml` 含 `pages_build_output_dir`，故全栈本地用 `pages dev dist`（需先 `npm run build` 生成 dist 供 `server` 提供静态资源；UI 走 vite 热更新）。
