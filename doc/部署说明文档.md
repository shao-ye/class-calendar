# 孩子每日课程记录日历 —— 部署说明文档

> 配套：[需求设计文档](需求设计文档.md)、[项目架构文档](项目架构文档.md)、[开发设计文档](开发设计文档.md)
>
> 本项目**已部署上线**于 `https://class.shaoruiyi.com`。本文档既记录已完成的部署，也作为**重建 / 迁移 / 日常更新**的操作手册。

## 一、前置条件

- Node.js ≥ 18（开发用 22），npm
- 一个 Cloudflare 账号，且域名 `shaoruiyi.com` 已托管在该账号
- wrangler 登录：本仓库已随依赖安装 wrangler；通过环境变量 `CLOUDFLARE_API_TOKEN` 鉴权（或 `wrangler login` 交互登录）

### API Token 所需权限
若用自定义 API Token，需包含：

| 类型 | 权限 |
| --- | --- |
| Account | Cloudflare Pages → Edit |
| Account | D1 → Edit |
| Account | Workers R2 Storage → Edit |
| Account | Workers Scripts → Edit（定时 Worker） |
| Account | Account Settings → Read |
| User | Memberships → Read |
| Zone（shaoruiyi.com） | DNS → Edit、Zone → Read |

> ⚠️ Token 等同账号凭据；交给他人使用后请及时 Roll/回收。

## 三、首次部署步骤（从零重建）

> 已部署环境无需重跑；以下为重建/迁移流程。命令均在 `class/` 目录执行。

### 1) 安装依赖
```bash
npm install
```

### 2) 创建 D1 并写入 database_id
```bash
npx wrangler d1 create class_calendar
# 把输出的 database_id 填入 wrangler.toml 的 [[d1_databases]].database_id
```

### 3) 远程建表与种子用户
```bash
npm run db:init:remote     # wrangler d1 execute class_calendar --remote --file=./schema.sql
npm run db:seed:remote     # 默认用户 parent / 123456
```

### 4) 创建 R2 桶（wrangler.toml 已含 PHOTOS 绑定）
```bash
npx wrangler r2 bucket create class-photos
```

### 5) 创建 Pages 项目并设置会话密钥
```bash
npx wrangler pages project create class-calendar --production-branch master
# 设置随机 SESSION_SECRET（值不回显）
SECRET=$(node -e "console.log(require('crypto').randomBytes(32).toString('hex'))")
echo "$SECRET" | npx wrangler pages secret put SESSION_SECRET --project-name class-calendar
```

### 6) 构建并部署
```bash
npm run deploy     # = vite build && wrangler pages deploy dist --project-name class-calendar
```
部署成功后会得到 `https://class-calendar.pages.dev`。

### 7) 绑定自定义域名 `class.shaoruiyi.com`
当前 wrangler v3 无 `pages domain` 子命令，用 API 完成（`$CLOUDFLARE_API_TOKEN` 来自环境变量）：
```bash
ACC=<ACCOUNT_ID>
# 7a. 注册自定义域名到 Pages 项目
curl -s -X POST "https://api.cloudflare.com/client/v4/accounts/$ACC/pages/projects/class-calendar/domains" \
  -H "Authorization: Bearer $CLOUDFLARE_API_TOKEN" -H "Content-Type: application/json" \
  -d '{"name":"class.shaoruiyi.com"}'
# 7b. 在 shaoruiyi.com 区域加 CNAME（代理开启）
Z=<ZONE_ID>
curl -s -X POST "https://api.cloudflare.com/client/v4/zones/$Z/dns_records" \
  -H "Authorization: Bearer $CLOUDFLARE_API_TOKEN" -H "Content-Type: application/json" \
  -d '{"type":"CNAME","name":"class","content":"class-calendar.pages.dev","proxied":true}'
```
证书几十秒至几分钟自动签发；状态短暂为 `pending` 但通常已可 HTTPS 访问。

### 8) 部署定时同步 Worker 并初始化假日数据
```bash
# 部署（绑定同一个 D1，注册 Cron）
npx wrangler deploy -c worker-holidays/wrangler.toml
# 设置手动触发保护口令
KEY=$(node -e "console.log(require('crypto').randomBytes(16).toString('hex'))")
echo "$KEY" | npx wrangler secret put SYNC_KEY -c worker-holidays/wrangler.toml
# 立即灌一次数据（也可登录后在「我的」页点「同步法定假日」）
curl -s "https://class-holidays-sync.<你的子域>.workers.dev/?key=$KEY"
```

## 四、日常更新（改代码后重新发布）

> Pages 项目生产分支为 `master`，与本仓库默认分支一致。`wrangler pages deploy` 会用当前 git 分支名作为部署分支：在 `master` 分支上 `npm run deploy` 直达生产域名 `class.shaoruiyi.com`；在其他分支部署则只生成预览环境（`<分支>.class-calendar.pages.dev`），不影响生产。若需在非 `master` 分支发布到生产，显式加 `--branch master`。

```bash
# 前端 / Functions 改动
npm run deploy

# 数据库结构变更（schema.sql 用 IF NOT EXISTS，新增表/列需单独 ALTER）
npx wrangler d1 execute class_calendar --remote --command "ALTER TABLE ..."

# 定时 Worker 改动
npx wrangler deploy -c worker-holidays/wrangler.toml
```

## 五、配置项

| 配置 | 位置 | 说明 |
| --- | --- | --- |
| `SESSION_SECRET` | Pages secret | 会话签名密钥（必须，生产用随机值） |
| `SYNC_KEY` | Worker secret | 保护 Worker 的手动同步入口 |
| `HOLIDAY_API_URL` | （可选）Worker / Functions 变量 | 假日数据源模板，含 `{year}`；缺省 jsdelivr 镜像 |
| Cron 频率 | `worker-holidays/wrangler.toml` 的 `[triggers].crons` | 当前 `0 18 28 * *`（每月，北京时间次日 02:00） |

## 七、修改默认密码（强烈建议）

线上初始口令 `parent/123456` 为公网弱口令，务必更换：
```bash
# 生成新哈希
node scripts/gen-hash.mjs '你的新密码'
# 用输出的哈希更新远程用户
npx wrangler d1 execute class_calendar --remote \
  --command "UPDATE users SET password_hash='粘贴哈希' WHERE username='parent';"
```

## 八、常见问题

- **部署后报数据库未绑定 / 接口 500**：确认 `wrangler.toml` 含 `pages_build_output_dir = "dist"`，否则 Pages 部署会忽略 wrangler.toml（D1/R2 绑定不生效）。
- **`wrangler pages dev` 报 "Specify either a directory OR a proxy command"**：因配置了 `pages_build_output_dir`，本地全栈请用 `npm run server`（`pages dev dist`）+ `npm run ui`（vite 代理）两进程，不要用 `-- 命令` 形式。
- **假日同步 count=0**：当年数据源可能尚未发布；可换 `HOLIDAY_API_URL` 或手动 `INSERT` 兜底。
- **权限报错 10000 Authentication error**：API Token 缺少对应权限，按第一节补齐。
