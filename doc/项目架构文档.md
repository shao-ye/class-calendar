# 孩子每日课程记录日历 —— 项目架构文档

> 配套：[需求设计文档](需求设计文档.md)、[开发设计文档](开发设计文档.md)、[部署说明文档](部署说明文档.md)

## 一、总体架构

应用基于 **Cloudflare 全家桶**，单一前端 SPA + 同仓后端 Functions + D1/R2 存储，外加一个独立的定时同步 Worker。

```
                          浏览器（手机优先）
                                │ HTTPS
        ┌───────────────────────▼───────────────────────┐
        │     Cloudflare 边缘（class.shaoruiyi.com）      │
        │                                                │
        │   Pages 静态资源(SPA)        Pages Functions    │
        │   ─ Vue3 + Vite 构建产物      ─ /api/* 路由      │
        │     (index.html + assets)    ─ _middleware 鉴权 │
        └───────────────┬───────────────────┬────────────┘
                        │                   │
                  ┌─────▼─────┐       ┌─────▼─────┐
                  │   D1      │       │    R2     │
                  │ (SQLite)  │       │ class-    │
                  │ 业务数据  │       │ photos    │
                  └─────▲─────┘       └───────────┘
                        │ 定时 upsert
                  ┌─────┴───────────────┐
                  │ Worker              │  Cron: 0 18 28 * *
                  │ class-holidays-sync │  → 拉取 holiday-cn → 写 holidays 表
                  └─────────────────────┘
                        外部数据源：holiday-cn (jsdelivr CDN)
```

## 二、组件职责

| 组件 | 类型 | 职责 |
| --- | --- | --- |
| **Pages 静态站** | Cloudflare Pages | 托管 Vue SPA 构建产物（`dist/`），自带 CDN + HTTPS |
| **Pages Functions** | Workers 运行时 | `functions/api/**` 提供 REST API；`_middleware` 统一鉴权与用户隔离 |
| **D1** | SQLite 托管库 | 用户、认证、课程、课时包、记录、照片元数据、假日 |
| **R2** | 对象存储 | 照片二进制文件（元数据存 D1） |
| **class-holidays-sync** | 独立 Worker | Cron 定时把法定假日同步进 D1（Pages 不支持 Cron 故单列） |

## 三、线上资源清单

| 资源 | 名称 / 标识 |
| --- | --- |
| Cloudflare 账号 ID | `<ACCOUNT_ID>` |

| Worker 机密 | `SYNC_KEY`（手动触发同步的保护口令） |

## 四、运行时绑定（bindings）

Pages Functions（`wrangler.toml`）：
- `DB` → D1 `class_calendar`
- `PHOTOS` → R2 `class-photos`
- `SESSION_SECRET` → Pages secret

定时 Worker（`worker-holidays/wrangler.toml`）：
- `DB` → 同一个 D1 `class_calendar`
- `SYNC_KEY` → Worker secret
- 可选 `HOLIDAY_API_URL` → 数据源模板（缺省内置 jsdelivr 镜像）

## 五、请求与鉴权流转

1. 浏览器访问 `class.shaoruiyi.com` → 边缘返回 SPA。
2. 路由守卫调用 `GET /api/auth/me` 判断登录态；未登录跳 `/login`。
3. 登录 `POST /api/auth/login`：校验 D1 用户密码（PBKDF2）→ 下发 **HMAC 签名的 `session` Cookie**（HttpOnly，30 天）。
4. 之后所有 `/api/*` 请求经 `functions/api/_middleware.js`：放行登录/登出，其余校验会话并把 `user_id` 注入 `context.data.uid`，各接口据此按用户隔离数据。
5. 图片 `GET /api/photos/:id` 同样走鉴权（`<img>` 同源请求自动带 Cookie），从 R2 读流返回。

## 六、数据流要点

- **课时核算**：课时包"已用"不冗余存储，而是 `opening_used + Σ(到课记录的 consumed_hours)` 实时算；"剩余"= 总课时 − 已用。续费时旧包置 `done`、新包 `active`。
- **计费与花费**：课程三种计费方式（`bill_mode`：课时包 / 单节付费 / 不计费）。花费按到课记录、按周/月统一折算——单节付费记本节 `fee`，课时包按"扣课时 × 包均价"换算；统计另列"累计已购课包"。
- **历史回填**：把课时包"期初已用"按上课星期从今天往前补成过往记录（幂等、跳假日与已有记录、不排未来），便于补齐历史并统计周期花费。
- **自动排课**：纯前端投影——读 `holidays`（按年）+ 课程的 `weekdays/skip_holiday/track_hours` + 当前生效包剩余，从今天向后生成"计划课"，不落库；"确认到课"才写 `records`。
- **假日同步**：外部数据 → Worker（Cron）/ 手动接口 → upsert `holidays`（前端只读）。

## 七、目录结构（顶层）

```
class/
├── doc/                     # 文档（本文档所在）
├── prototype/               # 早期可点击 HTML 原型

```

## 八、非功能与约束

- **成本**：全部在 Cloudflare 免费额度内（论证见需求文档 5.2）。
- **安全**：HTTPS 全程；会话 HttpOnly Cookie；数据按 `user_id` 隔离；密码 PBKDF2 加盐哈希。
- **可用性**：边缘 CDN，移动端优先布局。
- **已知约束**：Pages Functions 无 Cron（已用独立 Worker 解决）；Cloudflare 免费版边缘不在中国大陆，国内访问速度受限。
